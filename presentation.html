Project Overview & Goals

  • Intuitive Income/Expense CRUD: Record daily financials with custom categories, dates, and annotations.

Core Objectives:

To create a high-fidelity MERN application demonstrating production-level CRUD endpoints, JWT security, custom SVG drawing methods, and offline database fallback mechanics.

Dual-Mode Database Connection

  • Dynamic Failover Protection: Attempts Mongoose MongoDB connection. If absent, shifts to JSON mode automatically.
  • Mongoose APIs: Normal Mongoose schemas mapping users and transaction records when connected to MongoDB.
  • Local Store Fallback: Creates a localized JSON store (`dbStore.js`) managing files under `server/data/store.json`.
  • Unified Interface: Models export methods wrapping queries. Controllers call the same API in both modes!

Backend Endpoints & Security

  • Authentication Endpoints: Registry (`POST /register`), session sign-in (`POST /login`), profile load (`GET /profile`).
  • Transaction Controllers: CRUD paths (`GET /`, `POST /`, `PUT /:id`, `DELETE /:id`) checking ownership.
  • Password Protection: Uses `bcryptjs` hashing in both local offline and MongoDB database modes.
  • JWT Authorization Headers: Validates `Bearer ` in header middleware mapping users to requests.
